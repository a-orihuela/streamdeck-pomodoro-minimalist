# Internationalization (i18n)

The Stream Deck SDK provides built-in support for localizing plugins into multiple languages. Language resource files are stored as JSON inside the `*.sdPlugin` directory, and the SDK exposes a `translate()` API for accessing custom localization strings at runtime.

## Supported Languages

The SDK supports eight languages, each corresponding to a JSON file:

| Language              | File           | Notes                          |
|-----------------------|----------------|--------------------------------|
| Chinese (Simplified)  | `zh_CN.json`   |                                |
| Chinese (Traditional) | `zh_TW.json`   | Available from Stream Deck 6.8 |
| German                | `de.json`      |                                |
| English               | `en.json`      |                                |
| French                | `fr.json`      |                                |
| Japanese              | `ja.json`      |                                |
| Korean                | `ko.json`      |                                |
| Spanish               | `es.json`      |                                |

## Plugin File Structure

Language JSON files are placed directly inside the `*.sdPlugin` directory alongside the manifest:

```
.
├── *.sdPlugin/
│   ├── bin/
│   ├── imgs/
│   ├── logs/
│   ├── ui/
│   │   └── increment-counter.html
│   ├── de.json
│   ├── en.json
│   ├── es.json
│   ├── fr.json
│   ├── ja.json
│   ├── ko.json
│   ├── manifest.json
│   ├── zh_CN.json
│   └── zh_TW.json
├── src/
│   ├── actions/
│   │   └── increment-counter.ts
│   └── plugin.ts
├── package.json
├── rollup.config.mjs
└── tsconfig.json
```

## Localized Resources

Localization resource files serve two purposes:

1. **Override manifest strings** — values displayed throughout Stream Deck (e.g. plugin name, action names, tooltips).
2. **Provide custom strings** — arbitrary key/value pairs your plugin and property inspector can look up at runtime.

> **Note:** `sdpi-components` handles property inspector resources differently — localization strings are placed directly in the HTML files rather than in the language JSON files.

If no localization file is provided for the active language, Stream Deck falls back to the values in `manifest.json`.

## Localizable Manifest Strings

The following fields in `manifest.json` can be overridden by a language file:

### Root Level

- `Name`
- `Description`

### Actions

- `Actions[].Name`
- `Actions[].Tooltip`

### Encoder Trigger Descriptions

- `Actions[].Encoder.TriggerDescriptions.LongTouch`
- `Actions[].Encoder.TriggerDescriptions.Push`
- `Actions[].Encoder.TriggerDescriptions.Rotate`
- `Actions[].Encoder.TriggerDescriptions.Touch`

### States

- `Actions[].States[].Name`

## Manifest Example

The following `manifest.json` excerpt shows English strings for a volume controller plugin:

```json
{
    "Name": "Volume Controller",
    "Description": "Take control of your audio volume",
    "Actions": [
        {
            "UUID": "com.example.volume.adjust",
            "Name": "Volume control",
            "Tooltip": "Control your volume",
            "States": [
                {
                    "Name": "Unmute"
                },
                {
                    "Name": "Mute"
                }
            ],
            "Encoder": {
                "TriggerDescription": {
                    "LongTouch": "Mute",
                    "Push": "Toggle mute",
                    "Rotate": "Adjust",
                    "Touch": "Stummschaltung umschalten"
                }
            }
        }
    ]
}
```

## Language File Structure

### Overriding Manifest Strings

A language file mirrors the structure of `manifest.json` for overridable fields. Action-level overrides are keyed by the action's `UUID`. The following `de.json` provides German translations for the volume controller manifest above:

```json
{
    "Name": "Lautstärkeregler",
    "Description": "Übernehmen Sie die Kontrolle über Ihre Audiolautstärke",
    "com.example.volume.adjust": {
        "Name": "Lautstärkeregelung",
        "Tooltip": "Kontrollieren Sie Ihre Lautstärke",
        "States": [
            {
                "Name": "Stummschaltung aufheben"
            },
            {
                "Name": "Stumm"
            }
        ],
        "Encoder": {
            "TriggerDescription": {
                "LongTouch": "Stumm",
                "Push": "Stummschaltung umschalten",
                "Rotate": "Anpassen",
                "Touch": "Stummschaltung umschalten"
            }
        }
    }
}
```

### Adding Custom Strings

Arbitrary custom strings can be added under a `"Localization"` key in any language file. These strings are accessible via the `streamDeck.i18n.translate()` API:

```json
{
    "Name": "Lautstärkeregler",
    "Description": "Übernehmen Sie die Kontrolle über Ihre Audiolautstärke",
    "com.example.volume.adjust": {
        "Name": "Lautstärkeregelung",
        "Tooltip": "Kontrollieren Sie Ihre Lautstärke",
        "States": [
            {
                "Name": "Stummschaltung aufheben"
            },
            {
                "Name": "Stumm"
            }
        ],
        "Encoder": {
            "TriggerDescription": {
                "LongTouch": "Stumm",
                "Push": "Stummschaltung umschalten",
                "Rotate": "Anpassen",
                "Touch": "Stummschaltung umschalten"
            }
        }
    },
    "Localization": {
        "More info": "Weitere Informationen",
        "Save": "Speichern",
        "Reset": "Zurücksetzen"
    }
}
```

## API: `streamDeck.i18n.translate()`

Looks up a custom localization string defined under the `"Localization"` key in a language file.

### Signature

```typescript
streamDeck.i18n.translate(key: string, language?: string): string
```

### Parameters

| Parameter  | Type     | Required | Description                                                                  |
|------------|----------|----------|------------------------------------------------------------------------------|
| `key`      | `string` | Yes      | The localization key to look up (must exist under `"Localization"` in the language file). |
| `language` | `string` | No       | BCP 47-style language code (e.g. `"de"`, `"es"`). Defaults to the active Stream Deck language. |

### Return Value

Returns the localized `string` for the given key and language. If the key is not found in the specified language file, the function returns the key itself unchanged.

### Examples

```typescript
import streamDeck from "@elgato/streamdeck";

// Uses the active Stream Deck language (assumed English here)
streamDeck.i18n.translate("More info");
// => "More info"

// Explicitly request German
streamDeck.i18n.translate("More info", "de");
// => "Weitere Informationen"

// Request Spanish — es.json has no "Localization" entry for this key
streamDeck.i18n.translate("More info", "es");
// => "More info"  (key returned unchanged when translation not found)
```

## Localization Resolution Order

When resolving a localization string, Stream Deck follows a priority order:

1. The language file matching the active Stream Deck language (e.g. `de.json`).
2. The English fallback (`en.json`), if the active language file does not define the key.
3. The raw manifest value, if no language file defines the key.

> **Note:** A language JSON file overrides manifest content for all matching keys, even when the active language is the same as the manifest's base language.
